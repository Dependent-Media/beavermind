# Sample plans

Plan JSON files in this directory are real BeaverMind outputs — generated by Claude using the bundled fragment library, then captured verbatim. Drop them into a fresh BeaverMind install via:

```bash
wp eval-file wp-content/plugins/beavermind/bin/apply-sample-plan.php docs/samples/landing-beavermind.json
```

(See `bin/apply-sample-plan.php` for the runner.)

## Files

| File | Source | Notes |
|---|---|---|
| `landing-beavermind.json` | A self-referential brief asking Claude to design a landing page for BeaverMind itself, including the "Five inputs / curated fragment library / iterative refinement / brand-aware / bulk variants / multi-page" claim grid | Theme: warm craftsman mahogany (`#7c2d12`) derived from the brief's color request. 5 fragments. ~22s round trip. |

## How a sample is generated

The `tests/generate_self_landing.php` script runs `Planner::plan()` with a fixed brief, strips the per-run `usage` block from the response, and writes the JSON here. It also applies the plan locally so you can take a screenshot of the rendered page for the README.

To regenerate or add new samples:

```bash
ssh <site> 'bash -lc "cd ~/httpdocs && wp eval-file wp-content/plugins/beavermind/tests/generate_self_landing.php"'
scp <site>:httpdocs/wp-content/plugins/beavermind/docs/samples/*.json docs/samples/
```
